Lab 2 · First boto3 inventory script

Track: Python Automation · Typical duration: 60–90 minutes · Level: Intermediate

Python boto3 EC2 & S3 inventory Foundation for later automation

1. Objective

Implement a Python script using boto3 that lists EC2 instances and S3 buckets, returning a concise view of key metadata (IDs, names, regions, tags).

2. Context

Before more advanced automation is introduced, engineering teams benefit from a reliable view of “what actually exists” in their environments. This lab establishes that baseline.

3. Prerequisites

  • Lab 1 completed (IAM user / profile ready).
  • Python 3.x installed with the AWS SDK for Python (boto3).
  • Configured credentials/profile for the lab account.

4. High-level steps

  • Verify Python and boto3 installation.
  • Write a script that lists EC2 instances and prints selected fields.
  • Extend the script to list S3 buckets and basic properties.
  • Optionally enrich results with tags (Owner, Environment, Application).
  • Run the script and capture the output for later reference.

5. Expected outputs

  • A Python file (for example, inventory.py) stored locally or in a repository.
  • Example console output demonstrating EC2 and S3 inventory.
  • Short notes on how this inventory could be extended in future work.
This lab is frequently reused as the starting point for tagging enforcement, reporting and cost-awareness workflows in later sessions.
