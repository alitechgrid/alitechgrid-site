Lab 1 · Account hygiene & IAM user setup

Track: Cloud & IAM Foundations · Estimated time: 60–90 minutes · Mode: guided lab

Prereq: basic AWS familiarity Prereq: AliTechGrid Module 1 intro Safe to run in lab / student account

1. Lab overview

In this lab, students harden a fresh AWS account for classroom use and create a dedicated IAM user with MFA. The goal is to separate the root account from day-to-day work and establish a safe baseline for all later labs.

2. Learning objectives

  • Explain the risks of using the root account for everyday work.
  • Configure IAM users, groups and policies for a student lab environment.
  • Enable MFA on the IAM user and verify sign-in works.
  • Record the final sign-in URL and credentials in a secure location.

3. Prerequisites

  • Access to a clean AWS account intended for labs (not production).
  • Ability to sign in as the root account with email + password + MFA (if enabled).
  • Optional: authenticator app installed (phone or desktop).

4. High-level steps (student view)

Use this sequence in the lab PDF / LMS instructions. You can expand each step with screenshots and exact console paths.

  • Review the AWS console layout and locate the IAM service.
  • Create an IAM group for “Students” with a managed policy (e.g., PowerUserAccess or custom).
  • Create an IAM user for yourself (student-X) and add it to the group.
  • Set a strong initial password and require password reset on first login (if appropriate).
  • Sign out as root, then sign in with the new IAM user.
  • Enable MFA for the IAM user and test sign-in one more time.
  • Copy the account-specific sign-in URL and store it in your lab notes.
Instructor note: keep the detailed screenshots, exact policy names and answer key in your private DOCX / PDF. The HTML page is meant as a public-safe summary that links back to the full materials in your LMS or local evaluation app.

5. Expected student deliverables

  • Screenshot of the IAM user page showing MFA is enabled.
  • Screenshot or copy of the account-specific sign-in URL.
  • Short written explanation: “Why is it dangerous to use the root account for labs?”

6. Assessment (summary)

  • Correct setup (40%) – IAM user exists, in correct group, MFA enabled.
  • Evidence (30%) – screenshots / notes uploaded correctly to LMS.
  • Understanding (30%) – short response demonstrates awareness of IAM best practices.
