Modules & topics

Use the modules below as your official AliTechGrid syllabus. You can turn this into a course outline, LMS module list, or self-study checklist.

Module 1

AWS Core & Identity

Goal: understand accounts, regions, IAM and basic security.
  • What is AWS? Regions, AZs and global infrastructure.
  • Accounts, root user, best-practice separation for learning vs. production.
  • IAM users, groups, roles and policies – the mental model.
  • Hands-on: create a least-privilege IAM user for labs.
  • Hands-on: enable MFA and test login with the new user.
Module 2

Python + boto3 Automation

Goal: control AWS from code instead of the console.
  • Setting up credentials for boto3 safely (never hard-coding keys).
  • Listing EC2 instances, S3 buckets and IAM users from Python.
  • Filtering resources by tag, region and state.
  • Hands-on: script to tag EC2 instances with “Owner” and “Environment”.
  • Hands-on: generate a simple inventory report as CSV.
Module 3

Serverless APIs with Lambda

Goal: deploy logic without servers and expose it as an HTTPS API.
  • Lambda basics: runtime, handler, memory, timeout.
  • Event-driven architecture and triggers.
  • API Gateway HTTP API vs REST API (high level).
  • Hands-on: deploy a simple Python Lambda function.
  • Hands-on: connect API Gateway + Lambda and call it from Postman or browser.
Module 4

DynamoDB & CloudWatch

Goal: combine data storage with monitoring & logs.
  • DynamoDB basics: tables, keys, read/write capacity.
  • Hands-on: create a table for storing API requests or student records.
  • Read/Write examples from Lambda using boto3.
  • CloudWatch logs & metrics: where to look when things break.
  • Hands-on: create an alarm on Lambda errors.
Module 5

AI Workloads on AWS

Goal: connect AI services to real cloud workflows.
  • Overview of AI services: Comprehend, Rekognition, Transcribe, Bedrock.
  • Hands-on: text sentiment analysis with Comprehend or Bedrock model.
  • Use S3 as an input/output bucket for AI jobs.
  • Option: build a simple chatbot endpoint with Lambda + Bedrock.
  • Discussion: responsibility, cost control, and safe AI usage in the cloud.
Module X

Projects & Portfolio (optional)

Goal: turn what you learned into something you can show.
  • Pick a small scenario (ticketing system, student tracker, IoT logs, etc.).
  • Design the minimal architecture on paper.
  • Implement: IAM, storage, compute, monitoring, automation.
  • Document: short README + architecture diagram.
  • Present: short recorded demo linked from YouTube or GitHub.

Hands-on lab pack (10 labs)

The labs below match the AWS Automation with Python video series. You can host PDF/DOCX lab manuals in your repo or LMS and link them from here.

  1. Lab 1 – AWS account hygiene & IAM user setup Create a dedicated lab IAM user with MFA, console login and CLI access.
  2. Lab 2 – First boto3 script: list EC2 & S3 resources Install boto3, configure credentials and print out a simple inventory.
  3. Lab 3 – Tagging automation for EC2 and S3 Write a Python script that enforces “Owner” and “Environment” tags.
  4. Lab 4 – Deploying a basic Lambda function (Python) Create, test and monitor a Lambda function that returns JSON.
  5. Lab 5 – Lambda + API Gateway HTTP API Expose your Lambda as an HTTPS endpoint and test from browser/Postman.
  6. Lab 6 – DynamoDB CRUD from Lambda Build and query a simple DynamoDB table using Python code.
  7. Lab 7 – CloudWatch logging & alarms Investigate logs, create an error alarm and break/fix Lambda on purpose.
  8. Lab 8 – Simple cost & usage monitor Use AWS tools or APIs to see where money would be spent in this architecture.
  9. Lab 9 – Text analysis with AWS AI Send sample text to Comprehend/Bedrock and interpret the output.
  10. Lab 10 – Mini-project consolidation Combine IAM, Lambda, API Gateway, DynamoDB and AI into a tiny end-to-end demo.
