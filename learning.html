Learning hub & program structure

This page outlines how the AliTechGrid catalog fits together: core tracks, modules, and the 10-lab pack that supports hands-on delivery for individuals and organisations.

Tracks & focus areas

Each track can stand alone or be combined into a broader adoption or upskilling initiative.

Track 1 · Cloud Foundations & Governance

AWS accounts, IAM & controls

Build a clear, sustainable foundation for AWS usage across teams: account strategy, identity and access patterns, and operational guardrails.

  • Secure account and identity baselines.
  • Tagging, environments and ownership models.
  • Cost visibility at a practical level.
Track 2 · Python for Cloud Automation

Automation for practitioners

Enable engineers to replace ad-hoc console work with Python-based workflows that can be reviewed, versioned and reused.

  • Python essentials (targeted at cloud engineers).
  • boto3 patterns for inventory, tagging and reporting.
  • Patterns for scheduled jobs and operational tasks.
Track 3 · Serverless & AI Workloads

Modern workloads, realistic scope

Build small, well-scoped workloads that demonstrate serverless, managed data services and AI capabilities without over-engineering.

  • Lambda and API Gateway for simple APIs.
  • Data persistence options and observability.
  • AI services integrated into real use cases.

Modules at a glance

Modules are building blocks. They can be delivered sequentially, as standalone sessions, or blended into organisation-specific programs.

Module 1

Cloud & IAM Foundations

Orientation to the AWS environment, identity models, account hygiene and secure baselines.

  • AWS global infrastructure, regions and services.
  • Identity, authentication and authorisation fundamentals.
  • Hands-on baseline suitable for later automation work.

Backed by early labs in the lab pack (for example, IAM hygiene and account setup).

Module 2

Python + AWS Automation

Core Python capabilities for cloud engineers, combined with AWS automation scenarios that matter in real environments.

  • Python recap with a focus on practical tasks.
  • boto3 for inventory, tagging and reporting.
  • Patterns that can plug into CI/CD or operations processes.
Module 3

Serverless APIs & Data

Create small but real services with Lambda, API Gateway and data services such as DynamoDB, with observability included from the start.

  • API design at a pragmatic level.
  • Serverless execution models and security considerations.
  • Logging, metrics and basic alerting.
Module 4

AI & Automation Workloads

Introduce AI/ML services and automation patterns that use AWS capabilities in a controlled, outcome-driven way.

  • Patterns for AI APIs combined with automation.
  • Evaluation of value, cost and operational impact.
  • Examples suitable for demos, internal showcases or portfolios.

A more detailed view of module content and outcomes is available on the Modules page.

Example learning journeys

These are reference patterns. Real engagements can be shorter, longer or adapted to specific teams and constraints.

Engineer-level program (4–6 weeks)

  • Cloud & IAM foundation sessions + initial labs.
  • Python automation modules focused on day-to-day tasks.
  • Introductory serverless and AI workloads with small demos.

Suitable for engineers moving into cloud roles or extending current responsibilities.

Team enablement / onboarding program

  • Compressed foundations for teams already using cloud.
  • Custom automation labs aligned to existing environments.
  • Short serverless or AI exercises relevant to internal use cases.

Can be aligned with internal platforms, compliance requirements or product roadmaps.
