Module roadmap · AliTechGrid

Modules & learning outcomes

A structured path through AWS foundations, Python automation and AI workloads on AWS – suitable for diploma courses, higher-ed programs and practitioner training.

4 core modules + optional capstone Each module: 1–2 weeks of teaching Backed by 10-lab pack

Core modules

These modules can be mapped directly to course weeks, LMS units or industry workshops. Each module has matching labs from the 10-lab pack.

Module 1 · Cloud & IAM Foundations

Track: Cloud & IAM · Level: Beginner → Practitioner
Labs: 1–2 Ideal: Week 1
  • AWS global infrastructure, regions, AZs and services.
  • Accounts, root user, IAM best practices and MFA.
  • IAM users, groups, roles and policies.
  • Hands-on: dedicated lab IAM user with MFA and sign-in URL.

Outcome: learners can log in safely and work in an AWS lab account without breaking production.

Module 2 · Python + boto3 Automation

Track: Python + AWS · Level: Intermediate
Labs: 2–4 Ideal: Weeks 2–3
  • Review of Python essentials: data types, control flow, functions, files.
  • Configuring AWS credentials and boto3 profiles.
  • Listing, tagging and filtering EC2 and S3 resources with Python.
  • Monitoring, simple cost-awareness and automation scripts.

Outcome: learners replace console clicking with reproducible Python scripts that can be scheduled or integrated into DevOps pipelines.

Module 3 · Serverless APIs & Data on AWS

Track: Serverless · Level: Intermediate
Labs: 5–7 Ideal: Weeks 3–4
  • Designing request/response payloads for simple APIs.
  • Lambda functions triggered by API Gateway.
  • DynamoDB basics: tables, keys, read/write patterns.
  • Logging and CloudWatch monitoring of serverless workloads.

Outcome: learners can ship a small but real serverless API backed by DynamoDB, ready for class demos or mini-projects.

Module 4 · AI & Automation Workloads on AWS

Track: AI & Automation · Level: Intermediate → Advanced
Labs: 8–10 Ideal: Weeks 4–5
  • Using boto3 to drive AWS AI services (text, image, basic ML).
  • Pattern: “Python script + AWS service + JSON result”.
  • Designing small AI-assisted mini-projects for students.
  • Discussing ethics, cost management and responsible use.

Outcome: learners can wire up Python code to AI-related AWS services and present simple end-to-end demos.

Optional capstone / skill project

You can bundle the modules and labs into a 2–4 week capstone for diploma programs or advanced courses.

Capstone · AI-powered Cloud Automation Project

Uses pieces from all modules · Ideal: end-of-term skill project
  • Students propose a small real-world scenario (e.g., monitoring, reporting, automation, data pipeline, lightweight AI workload).
  • Use IAM best practices from Module 1 and automation patterns from Module 2.
  • Optionally expose a simple API or dashboard based on Module 3.
  • Include at least one AI or analytics component from Module 4.

Deliverables: code repo, short report (2–3 pages), demo video or in-class presentation and self-assessment using your 5E / AI-proof evaluation model.
